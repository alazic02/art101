Lab 9 - JavaScript for the Web

Challenges

We faced quite a few challenges when completing this lab, mostly related to getting the new elements we created in the lab.js to properly append to the 'output' div in the index.html. Because there were no screenshots in the lab instructions, it was even more difficult to troubleshoot and fix.

Problems

In section, we went over a solution for appending the new elements that involved making the new1El and new2El variables assigned to the new elements into text nodes, which were then able to be appended onto the 'output' div. This worked because the error we were getting was an incompatible node type error. The HTML was not able to recognize the variables because they couldn't be read. However, the solution was a lot simpler than creating two new varibles.

In order to be able to append the variables properly without being turned into text nodes, we had to use the innerHTML method. By using this and writing our new elements as new1El.innerHTML = , it accesses the inside of the element we made (in this case a new paragraph) and added the text there. Doing it this way made it so that the element was now fully readible for the index.html, and so could be appended correctly to the 'output' div.
